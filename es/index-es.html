Lo que he hecho

Sitio web - carpeta

Cuando se trata del diseño y facilidad de uso, espero que cada sitio supere los estánderes básicos y no excluya a nadie.
Mientras estás aquí, puedes leer sobre mis proyectos. ¿Por qué no hacer click el logo de GitHub al lado de cada proyecto para ver el proceso completo? ¿O quizás quieres jugar a uno de los juegos?
Mi fiolosofía de diseño es : que sea sencillo y facil de leer.

Crucero

Hice un crucero interactivo y el usuario puede crear su propio itinerario.
Aprendí sobre test-driven development y la manipulación del DOM: construí unos objetos que pueden interactuar entre sí.
Fue la primera vez que tuve que obtener datos del usuario de un formulario; al principio fue un gran desafío.

Mascota virtual

¡Esto fue mi primer proyecto de javascript!
Mediante test-driven development, creé objetos javascript dinámicos con propiedades como la edad, el hambre y el estado físico. Luego, diseñé y creé una interfaz con la opción de 3 mascotas diferentes y animaciones únicas para cada una.
¡Para jugar a este juego, debes alimentar y jugar con la mascota para mantenerla con vida!

Pronóstico del tiempo

¡Este fue mi primer proyecto que creé con React!
Esto es una aplicación que puede extraer datos de pronóstico del tiempo desde un API. También agregué una función de búsqueda que permite al usuario buscar el tiempo en cualquier parte del Reino Unido.
Luego diseñé y creé una interfaz que (al menos yo creo) se ve muy bonita.

Biblioteca de música

Esto es un API de biblioteca de música construido con Node.js, Express.js y PostgreSQL.
Este API se creó de acuerdo con test-driven development y el marco de pruebas es Mocha, Chai y Supertest.
Este proyecto se hizo durante el módulo de back-end en Command Shift Bootcamp, y la experiencia me ha hecho sentir mucho más cómodo con los API.
